作为一个 MJJ，必不可少的项目那肯定是探针。毕竟有一个笑话这么说：一个程序员买了 20 台 VPS，朋友问他：“你拿这些干嘛？”他说：“跑项目。” “什么项目？” “监控其它 19 台 VPS 有没有掉线。”

**Komari** 就是一款不错的开源轻量级、自托管服务器状态监控工具（探针），旨在为个人开发者、站长和运维人员提供简单高效的服务器性能可视化方案。

## 🌟 核心特点

- **专注纯粹**：与功能日益臃肿的哪吒探针（Nezha）不同，Komari 放弃了 DDNS、内网穿透等非监控强相关的复杂功能，回归探针的本质。系统资源占用极低，哪怕是内存极小的“低配小鸡”也能轻松运行。在 256M 甚至 128M 的小鸡上你都可以扎针（Agent 客户端），不过扎针的时候我比较推荐关闭远程控制功能。
- **历史数据追踪**：原生支持最长达 30 天及以上的历史监控数据记录与查询，方便随时回溯服务器的运行状态和性能瓶颈。
- **部署极简**：提供简单直观的架构设计（服务端接收 + 轻量级 Agent 客户端推送），无论是 Docker 还是二进制安装都非常顺滑，维护成本极低。
- **开源与隐私**：前后端完全开源，用户可以 100% 掌控自己的数据隐私，无需依赖第三方云服务。
- **高颜值界面**：Web 面板设计直观美观，支持多种第三方主题（如 Purcarte 等，我就用的这个主题），数据展示（CPU、内存、流量、网络速度、运行时间等）一目了然。同时，你可以自定义站点的 `header` 和 `body`，为它添加一些美观又好玩的小组件。

## 🛠️ Docker Compose 部署示例

以下是我个人使用的 `docker-compose.yml` 示例（这里将端口绑定在了本地的 `127.0.0.1`，推荐配合 Nginx 反代使用以确保安全）：

```yaml
version: "3.8"

services:
  komari:
    image: ghcr.io/komari-monitor/komari:latest
    container_name: komari
    ports:
      - "127.0.0.1:25774:25774"
    volumes:
      - ./data:/app/data
    restart: unless-stopped
```

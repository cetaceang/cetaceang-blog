## 关于 RackNerd

**RackNerd 服务商简介**

**基本信息**  
RackNerd（RackNerd LLC）是一家美国 IaaS（基础设施即服务）提供商，成立于 **2019 年**，由数据中心行业资深人士 **Dustin B. Cisneros** 创立，总部位于美国加州 Rancho Cucamonga。  

**主营业务**  
RackNerd 主打企业级基础设施服务，产品线覆盖：  
- **KVM VPS** — 标准款 / AMD Ryzen 高性能款 / Windows VPS（最受欢迎）  
- **独立服务器** — Dedicated Servers
- **机柜托管**（Colocation）  
- **Private Cloud / DRaaS**（灾难恢复即服务）  
- **共享主机 / 转销商主机**  
- **定制解决方案** — 高级 DDoS 防护、负载均衡等  

**核心优势与卖点**  

1. **极致性价比**：作为与 CC (CloudCone) 齐名的廉价机，以“预算友好”著称，经常推出超低价促销（1C1G 的机器常年 10 刀/年，2C2G 大概 18 刀/年），配置包含 NVMe/SSD、大流量、1Gbps 端口，且多数促销套餐**续费同价**。  
2. **全球数据中心布局**：运营 **21 个数据中心**（覆盖 20 个位置），横跨北美、欧洲、亚洲：  
   - 北美重点：洛杉矶、圣何塞、达拉斯、芝加哥、西雅图、纽约、亚特兰大等  
   - 欧洲：阿姆斯特丹、伦敦、都柏林、斯特拉斯堡  
   - 亚洲：新加坡  
   *(兴许探针可以多点亮几个区域，笑)*
3. **华人用户友好**：支持**支付宝**支付，购买十分方便。

**总结**：这台 RN 加拿大机的 IP 质量中规中矩，风险评分处于中等水平，流媒体解锁能力尚可，顺带当个落地节点用也行；25 端口开放，可以用来搭建邮局，或许是一个不错的点。

**测试结果分析：**

- **三网回程路由**：
  - 电信：163
  - 联通：4837
  - 移动：CMI
- **国内延迟**（仅供参考，测试时间和网络状况会影响结果）：
  - 电信：多数地区在 230-350ms 左右，个别节点（如黑龙江）高达 679ms，整体偏高且波动大。
  - 联通：多数地区在 270-470ms 左右，部分节点出现 0 或超高延迟，稳定性较差。
  - 移动：多数地区在 220-410ms 左右，整体表现和电信差不多，个别节点（如河南）延迟飙到 748ms。
- **国际互联评价**：
  - 北美方向（洛杉矶、纽约）：延迟低（14-64ms），带宽大，同洲内表现优秀。
  - 东亚方向（东京 166ms、香港 215ms）：延迟偏高，带宽偏低且有重传，勉强能用。
  - 欧洲方向（伦敦 84ms、法兰克福 99ms）：延迟不错，带宽中等，日常使用没问题。
  - 大洋洲方向（悉尼 252ms）：延迟较高，带宽还行，不建议对延迟敏感的场景。

</details>

**总结**：还是那句话，RN、Dedirock、CC 这些 CCS 的下游，线路本身十分爆炸，能连上就算胜利。三网直连几乎不可用，毕竟一分价钱一分货，没有线路优化的机器就是这样。

### 硬件性能测试

<details>
<summary><strong>点击查看详细测试数据</strong></summary>

![硬件性能测试](https://imgbed.cetaceang.qzz.io/file/测评留档/1773154251611_rn2c2g基本信息.webp)

**测试结果分析：**

- **CPU**：Intel Xeon Gold 5118 @ 2.30GHz，2 核 2 线程
  - gb5 单核 469，多核 591。
  - 这个分数挺低的，单核比 Dedirock 都低。不过跑轻量服务肯定是够用了：New API、Komari 探针、一些 API 的网关，甚至跑一个窗口的浏览器自动化也可以胜任（亲身实测），姑且还算是能用。
- **内存**：1.9 GB
  - 读 15302 MB/s，写 11630 MB/s。内存性能中规中矩，符合这个价位的正常水平。
- **硬盘**：30G SSD
  - 4K 随机读：16.8 MB/s（4.3k IOPS）
  - 4K 随机写：25.1 MB/s（6.4k IOPS）
  - 顺序读写：能到 1-2.7 GB/s，大文件传输没压力。
  - 4K 随机性能一般，顺序读写还可以，IO 性能普通，不适合高负载数据库等场景。

</details>

**总结**：硬件方面没有什么惊喜也没有什么失望，CPU、内存和磁盘性能都是这个价位的正常水平。约 17 刀一年的机器，跑个反代、博客、轻量服务绰绰有余，跑单个窗口的浏览器自动化也还能运行。不建议跑太高负载的程序，不管是 CPU 高负载还是对硬盘 IO 要求高的应用。

## 实际使用场景与体验

如我前文所说，目前我在这个机器上部署了一些 AI 的中间网关，还跑了一个浏览器自动化的项目。实际上，一个窗口的浏览器自动化就会让这台机器的 CPU 峰值占用达到 80% 左右。不过好在任务都可以正常完成，没出现过浏览器崩溃之类的情况。虽然性能比较孱弱，但还是能满足个人使用的。

机器目前运行了 47 天，还没有出现过硬件问题、网络问题或其他稳定性问题，感觉稳定性尚可（不过有重要数据记得备份）。

如果你是小白，想要练习 Linux 命令、部署 Nginx、写 Nginx 配置或者折腾一些其它的事情，这台玩具机非常适合作为小白的第一台练手机器。

虽然我没有用过，不过我觉得大概率 OpenClaw 也是可以运行的。最近我的朋友使用和我相同款式的 Dedirock 运行 OpenClaw，带着浏览器，没出现什么机器导致的问题。那台机器都能运行，这台没理由不行。

## 总结：值得买吗？

> 📌 **快速判断**
>
> ✅ **推荐购买**：
> - 新手小白，需要一台机器练手学习 Linux
> - 想要高性价比机器部署轻量业务，不在意国内直连速度，觉得套个 CF 也不是不能用
> - 想玩 OpenClaw，需要一台便宜的、不小心龙虾删了机器上的东西也不心疼的玩具机
>
> ❌ **不推荐**：
> - 业务面向国内，对延迟和稳定性敏感
> - 需要高性能 CPU 或高 IO 的场景（如大型数据库、高并发业务）

### 我的建议

如果是小白，我觉得 RN 还是比 Dedirock 靠谱一点点，毕竟商家运营时间久，用户也多一点。不一定要选这款 18 刀左右的机器，RN 经常有 10 刀/年的特价机，可能更适合作为小白的第一台机器。RN 的机器经常被分配到 YouTube 送中 IP，如果需要解锁流媒体或者 Gemini 可能还要套 WARP。

日常使用没什么问题，虽然性能低，但是个人使用也不太会需要很强的性能，大概率也不会用到庞大的 MySQL 数据库。对于这个价格，MJJ 们常说一句话：“没有垃圾的产品，只有垃圾的价格。”RN 的产品完全对得起它的价格，属于绝对能用的范畴。

如果你已经是一个略有经验的 MJJ，也有着稳定的建站机，那为什么不看看一些更有趣的“灵车”呢？比如 Dedirock、狐蒂云之类的，又便宜又好玩，哈哈。
